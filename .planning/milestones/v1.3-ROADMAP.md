# Milestone v1.3 — Consistency & Code-Integrity Safeguards

**Scoped:** 2026-06-26 (via `/gsd:explore`)
**Status:** Scoped, not started
**Phases:** 10-11

## Problem

GSD's defining property is also a defining failure mode: work happens across many independent
agent sessions with **no shared memory** of each other. The result is **cross-session drift** —
each change is locally reasonable, but the codebase is globally inconsistent:

- **Semantic duplication** — session B re-implements logic session A already wrote, under a
  different name/shape.
- **Naming drift** — identifier casing and file-name conventions oscillate (`user_id` vs
  `userId` vs `uid`; `*.handler.ts` vs `*Controller.ts`).
- **Architectural split** — half the code uses DI / error-wrapping, half hits env directly /
  swallows errors.
- **Half-finished work** — placeholder `TBD`/`todo` returns and phantom (never-imported) exports
  left by interrupted sessions.

Two complementary tracks: **prevention** stops *new* drift at authoring time; **detection**
reconciles drift that *already happened*. Naming is catchable both ways; semantic duplication
is fundamentally detection-only (session B cannot know about session A at plan time).

## Evidence (VibeDrift evaluation, 2026-06-26)

Empirically evaluated the MIT-licensed VibeDrift CLI (v0.14.0, local-only, deterministic,
~30-60s/repo, no LLM) on 4 repos: gsd-plugin, get-shit-done-redux, LyricConductor, rentular.

**Validated, differentiated signal (worth building / cherry-picking):**
- Cross-session naming drift via **majority-vote convention derivation + entropy** — explicitly
  attributes oscillation to "different AI sessions." Best single idea to port into pattern-mapper.
- **Verb-vs-body intent mismatch** (name says `get`, body mutates).
- **Architectural-pattern split** (DI vs env; swallow vs throw vs wrap).
- **Phantom scaffolding** (CRUD-named exports never imported) + **placeholder returns** in prod code.
- **Consistency-relative security** (e.g. "33 mutating routes lack auth while the codebase uses
  auth elsewhere" — found on rentular).

**Caveats that shaped the design:**
- MinHash + LCS (structural Type-3), not embeddings — lighter, shallower than pure semantic.
- Noisy raw (50-line opinions, unreachable-after-return + unused-export false positives).
- **~38% of duplication findings on gsd-plugin were the INTENTIONAL CJS<->SDK dual resolver** — an
  intentional-duplication allowlist is mandatory.

**Decision:** adopt VibeDrift as an *optional external gate* (pinned, config-gated, local-only,
graceful-degrade) and *cherry-pick its heuristics* into GSD's own pattern-mapper + code-reviewer
so the capability survives even if a 2-day-old v0.x tool does not. Do **not** vendor; do **not**
hard-depend. Working notes: scratchpad `semantic-dup-research.md` + `vibedrift-evaluation.md`
(promote into `references/` during Phase 11 to make durable).

## Phase 10 — Prevention: convention/architectural conformance (plan -> execute -> review)

**Goal:** Stop a new file from introducing cross-session convention/architectural drift, using
the conventions the codebase already exhibits (derived, not hardcoded).

**Scope:**
- Extend `gsd-pattern-mapper` to emit an explicit **Conventions** section in `PATTERNS.md`:
  identifier casing, file-name casing, export style — derived by majority vote with an entropy
  signal, so the planner/executor get a named contract, not just "closest analog."
- Add a **conformance dimension** to `gsd-code-reviewer` that flags *changed* files deviating
  from the derived convention, plus a cheap **verb-vs-body** intent check and an
  **architectural-pattern split** check (DI vs env access, error-handling style). Deterministic,
  no heavy deps. Findings use recommended-fix framing (consistent with the resilience theme).

**Success criteria:**
- pattern-mapper writes a Conventions section (majority-vote + entropy) to PATTERNS.md.
- code-review flags a deliberately convention-violating changed file and passes a conforming one.
- No new runtime dependency; runs in the existing review path.

## Phase 11 — Detection: consistency sweep + optional VibeDrift gate

**Goal:** Surface existing cross-session drift repo-wide and gate the pre-1.0 release ceremony.

**Scope:**
- **Optional VibeDrift integration:** probe on PATH / `npx` (pinned version), `--local-only`,
  graceful-degrade when absent. Wire into `code-review` (on the diff) and into `audit-milestone`
  as a config-gated **pre-1.0 gate** (`--fail-on-score N`, recommended-fix framing). Ship an
  **intentional-duplication allowlist** (`.vibedrift` config) seeded with the CJS<->SDK dual
  resolver so it is suppressed but auditable.
- **`/gsd:scan --drift`** (lighter than `map-codebase`): run the sweep, route a ranked report.
- **Native fallback:** port the cheap heuristics (placeholder/phantom detection, MinHash
  semantic-dup) so the sweep degrades to GSD-native checks when VibeDrift is absent.
- "Less token overhead" only realized if detection **replaces** an LLM review pass, not adds to it.

**Success criteria:**
- `audit-milestone` runs an optional, config-gated integrity gate that the intentional dual
  resolver does not trip (allowlist verified, suppressions auditable via report).
- `/gsd:scan --drift` produces a ranked drift report on gsd-plugin.
- Graceful-degrade proven: sweep still runs (native checks) with VibeDrift uninstalled.

## Dependencies / notes

- Phase 10 and Phase 11 share the convention-extraction logic (majority-vote derivation); build it
  once in Phase 10, reuse in the Phase 11 native fallback.
- Privacy hard rule: never run VibeDrift `--deep` on private repos without explicit user consent
  (it ships function snippets to its cloud). Default and CI use `--local-only`.
- Requirements file (`v1.3-REQUIREMENTS.md`) and per-phase plans to be generated when the milestone
  starts (`/gsd:new-milestone` can formalize requirements, or plan phases directly via `/gsd:plan-phase 10`).
