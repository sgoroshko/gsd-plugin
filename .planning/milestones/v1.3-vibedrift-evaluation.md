# VibeDrift empirical evaluation (v0.14.0, local-only)

Ran `vibedrift scan . --local-only --json` on 4 repos. ~30-60s each, deterministic, no network, no LLM.

| repo | files | lines | composite | hygiene | findings |
|------|-------|-------|-----------|---------|----------|
| gsd-plugin | 438 | 107k | 70.3 | 91.9 | 481 |
| get-shit-done-redux | 697 | 191k | 78.3 | 96 | 663 |
| LyricConductor | 179 | 30k | 79.3 | 89 | 182 |
| rentular | 155 | 30k | 68.7 | 84 | 180 |

## Technique
"Code DNA" = MinHash on operation-sequences + LCS verification (Type-3 structural clones), NOT embeddings.
So it's lighter/faster than the embedding approach the research recommended, and catches structural near-dupes
but not deep Type-4 semantic equivalence. Good enough + zero infra.

## Differentiated, high-signal findings (the "ideas worth copying")
1. **Cross-session naming drift via majority-vote + entropy.** "type naming oscillates: 343 PascalCase, 80 other
   — likely from different AI sessions" (gsd-plugin); "function naming oscillates: 343 camelCase, 240 other"
   (LyricConductor). KEY IDEA: derive the convention from the codebase by majority vote, flag deviations — don't
   hardcode it. This is Track A, done well.
2. **Intent-clarity: verb-vs-body mismatch.** "44 functions where the leading verb doesn't match the body's
   control structure" — name says get/has but body mutates/loops. Novel, cheap, name-vs-behavior.
3. **Architectural drift.** "half use config via DI, half hit env directly"; "error swallowing vs throwing vs
   wrapping". Catches the exact two-sessions-two-patterns split GSD causes. Most GSD-relevant dimension.
4. **Phantom scaffolding / implementation-gap.** "CRUD-named exports never imported or routed"; "placeholder
   returns (TBD/todo) in production code". Catches half-finished AI work — ties to GSD's broken-session theme.
5. **Consistency-relative security.** "33 mutating routes lack auth while the codebase uses auth elsewhere"
   (rentular). High value: derived from the codebase's own norm, not a generic ruleset.
6. **Ops packaging worth emulating regardless:** `--fail-on-score N` exit codes, git pre-push hook, MCP server
   (`validate_change`, `find_similar_function`), JSON output. Release-gate ergonomics GSD's audit-milestone wants.

## Noise (suppress / don't copy raw)
- "functions exceed 50 lines" (opinion), "unreachable after return/throw" (early-return false positives),
  "exported symbols unused" (framework/library entry points = false positives), comment-density.
  These inflate counts; surfacing raw would erode trust.
- Duplication dominated by INTENTIONAL dual-impl without an allowlist: on gsd-plugin, ~129/338 dup findings
  (38%) are just the deliberate bin/lib (CJS) <-> sdk/src (TS) dual resolver. Allowlist is mandatory to be usable.

## "Less token overhead" — validated
Local mode is deterministic, ~30-60s, no LLM. If GSD delegates duplicate/naming/architectural/security-drift
detection to it and spends agent tokens only triaging its flagged findings, that's a real reduction vs today's
LLM-driven review. True only if it REPLACES an LLM pass, not if bolted on in addition.

## Verdict
Useful. Adopt as an OPTIONAL external gate + cherry-pick its best heuristics into GSD's own checks. Do NOT vendor
(v0.14.0, 21 stars, 35 commits, ~one maintainer, 2 days old). Do NOT hard-depend yet.
- Short term: optional external tool in `code-review` (on diff) + pre-1.0 `audit-milestone` gate (`--fail-on-score`),
  config-gated, pinned version, graceful-degrade when absent. Privacy: local-only, never `--deep` on private repos
  without explicit consent (it ships function snippets to their cloud).
- Long term: port the heuristic IDEAS (majority-vote convention derivation, verb-vs-body, architectural-pattern
  split, phantom/placeholder, consistency-relative security) into pattern-mapper + code-reviewer so capability
  survives even if VibeDrift doesn't mature.
