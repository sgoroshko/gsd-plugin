<objective>
Add a Flutter branch to GSD's UI design-contract pipeline (`/gsd:ui-phase`) without changing existing web behavior. Today `agents/gsd-ui-researcher.md`, `agents/gsd-ui-checker.md`, and `templates/UI-SPEC.md` are hardcoded to the web/shadcn stack (shadcn init gate, `components.json`, `.tsx`/`.jsx`/`.css` globbing, Registry Safety dimension). This task adds: (1) a stack-detection gate at the top of the researcher agent that hard-branches Flutter vs. web before any web-specific logic runs; (2) a new standalone `templates/UI-SPEC-flutter.md` with Material 3/Cupertino design tokens, Navigation Pattern, Touch & Platform Conventions, and Package Safety sections; (3) a mirrored stack-detection branch in the checker agent that scores the correct dimension set; (4) a light routing edit in `workflows/ui-phase.md` so the researcher receives the correct template path. The web path (React/Next.js/Vite via shadcn) must remain byte-for-byte unchanged — every edit to `agents/gsd-ui-researcher.md` and `agents/gsd-ui-checker.md` is a pure insertion, never a modification of existing web-path text.

Purpose: let a Flutter-project user run `/gsd:ui-phase`, give a text description and/or reference screenshots (optionally naming an app like Telegram/Instagram/Reddit as a style direction), and get a UI-SPEC.md with concrete Material 3-based design tokens -- without pixel-perfect precision and without the screenshot-based visual audit that stays web-only.

Output: `templates/UI-SPEC-flutter.md` (new file); additive edits to `agents/gsd-ui-researcher.md`, `agents/gsd-ui-checker.md`, `workflows/ui-phase.md`.
</objective>
